# CLAUDE.md — правила разработки

## Ветки и коммиты
- Ветка от `main`, имя `feature/<суть>`. В `main` напрямую не коммитим.
- Перед каждым коммитом — **pre-commit** (ruff формат+линт, тесты, скан секретов).
  Включить один раз: `pip install pre-commit && pre-commit install`
  (или гонять `pre-commit run` вручную). Любой шаг упал → **не коммитим**, чинимся.
- Не выходим за рамки задачи: объём изменений не должен заметно превышать запланированное.

## Сервисы
- Сервис = каталог `services/<имя>/` со своим `Dockerfile`. Новый сервис — просто новый
  каталог, пайплайн подхватит сам (имена нигде не зашиты).

## База знаний (онтология над кодом)
Документацию ведём как **онтологическую базу знаний**: каждый
документ — типизированный объект (`node_type`) со свойствами (frontmatter) и
**типизированными связями** на другие доки и на код (подход — как Palantir Ontology,
но над документацией). Так знание остаётся навигируемым и проверяемым линтером.
**Source of truth — markdown** под `docs/gitmark/` (остальной `docs/` — свободен для не-KB);
производный индекс поиска постоянно обновляется и в
git не хранится.

## Тесты
- Тесты лежат под `pyproject.toml` окружения; `pytest` находит их сам. Медленные/дорогие
  помечать `@pytest.mark.heavy`.
- CI — гейт по PR (затронутые тест-каталоги, `-m "not heavy"`). На мерж в `main` тесты не
  гоняются — только build изменённых образов и деплой на dev; полный прогон — вручную (`Full
  tests`) или на релиз. На push в feature CI не гоняется — до PR держит pre-commit. Детали — `TESTING.md`.

## Секреты
- Никогда не коммить ключи/токены/секретные `.env`.
- Несекретный конфиг → Environment **Variables**; 
  секреты → Environment **Secrets**.
